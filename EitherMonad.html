Why do you use GOTO?

Now, you thought: "What? I didn't!"

And go to write:


					throw new Exception(message);
Is throwing an exception a kind of goto?

Yes, because it let you BREAK THE FLOW

with goto:


					for (...) {
						for (...) {
							for (...) {
							...
								if (something)
									goto END_OF_LOOP;
							}
						}
					}					
					END_OF_LOOP:

The same with exception

	
					try
					{
						for (...) {
							for (...) {
								for (...) {
								...
									if (something)
										throw new Exception();
								}
							}
						}				
					}
					catch (Exception)
					{
					}

It's painful, but worked. And it's valid C# code.
